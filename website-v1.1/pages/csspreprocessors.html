Cascading Style Sheet (CSS)

CSS Preprocessors, Frameworks and Polyfills

  1. CSS Preprocessors
    • generates CSS using a custom language syntax that typically includes features that don't exist in pure CSS (e.g., variables, control flows, nesting, mixins, functions, etc.).
    • Example:
      1. SASS
        • curly braces can already be omitted.
        • Example:
          • @include
          • @mixin
          • @extend
      2. lesscss
      3. stylus
        • semi-colons can already be omitted.
    • Features:
      1. Variables
        • Variables store values of attributes which can be reused in any element minimizing the need to rewrite the same code everytime.
      2. Nesting
        • Nesting provides a hierarchical structure for developers similar to the HTML code which enables them to easily understand and read the CSS code.
      3. Mixins
        • Accepts parameters which can then be used to set the values of the attributes.
        • It uses @include to indlue the mixins within the CSS code.
      4. Extends
        • Used in sharing the same code with other elements which makes it easier rather than copying the entire code.
        • It uses @extend to extend the code within a block of CSS code.
  2. CSS Frameworks
    • provides functionality without having to write CSS.
    • Set of codes already provided by other users which other developers can use in developing their website.
    • Example:
      1. bootstrap
      2. foundation
      3. materialize
  3. Polyfills
    • allows users/developers to make use of the features of the browsers.
    • provides features that developers expect browsers to provide natively.
    • Example:
      1. Modernizer
      2. Selectivizer
