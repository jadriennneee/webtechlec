Cascading Style Sheet (CSS)

CSS Selectors

  • Selector
    • Structure used as a condition in a CSS Rule
    • It is used to specify the HTML elements we want to style.
  • Types of Selector:
    1. Simple Selector
      • It does not contain any combinators which means that it cannot contain other types of selectors or Universal selectors.
      • It starts with a type selector (HTML element) or a Universal Selector.
      • Example:
        h1 { text-align: center; }
      • Types of Simple Selectors:
        1. Universal Selector
          • It is denoted by an asterisk (*) symbol.
          • It means that it targets any element or component of the document.
          • Example:
            p { font-size: 20px };
        2. Type Selector
          • It specifies the element type specifically the HTML element allowing all elements that matches the indicated type Selector to be styled.
          • Example:
            p { color: red; }
        3. ID Selector
          • It is always introduced by the pound (#) sign.
          • This type of selector targets elements that has an attribute called 'id' and the values of the attributes matches the specified ID selector.
          • Example:
            #main { padding: 10px; }
        4. Class Selector
          • It is always introduced by the period (.).
          • It is somewhat similar to the ID Selector but it uses the values of the 'class' attribute instead.
          • Example:
            .container { margin-bottom: 10px; }
          • It can also target class attributes with 2 or more values.
          • Example:
            p.feature.quote { text-align: justfiy; }
        5. Attribute Selector
          • It is a generic way of associating design to the attributes of the document.
          • It is indicated by a square brackets ([]).
          • Example:
            img[alt] { border: 1px solid green }
          • Wildcards used in the selector:
            1. *=
              • This targets all attributes that contains the specified value.
              • It also means 'contains'.
              • Example:

                a[href *= google] this targets all href attributes that has a value containing the word 'google'

              • Example:
                a[href *= "google"]
            2. $=
              • This targets all attributes with values having asuffix similar to the value indicated in the selector.
              • Example:
                a[href $= ".png"]
            3. ^=
              • Instead of suffix, similar to No. 2, it targets all attrbutes with values having a prefix similar to the value indicated in the selector.
              • Example:
                a[href ^= https]
            4. |=
              • This targets attributes who has a value that has either an exact match or exact match with a hyphen/dash (-).
              • Example:
                p[lang |= en]
            5. ~=
              • This targets all attributes who has a value of one or more such that either of the values matches the indicated value for the selector.
              • Example:
                p[class ~= warn]
  • Selector Group
    • It is a comma-separated list of selectors to represent the union of all selectors which has a similar declaration.
  • Combinators
    • It is used to prepend/add another sequence of a selector.
    • It is a way of combining sequence of simple elements.
    • Types of Combinators:
      1. Descendant Combinator
        • Uses whitespace indicated by a tab, space,linefeed, carriagge return, etc. to combine selectors.
        • It selects elements which are direct descendants of the element indicated before it.
        • Example:

          div p targets all p elements which are direct descendants of the div element

      2. Child Combinator
        • Uses the greater than (>) sign to combine sequence of selectors.
        • It selects elements which are direct children of the element indicated before it.
        • Example:
          div > p targets all p elements who are direct children of the div element.
      3. Sibling Combinators
        1. Adjacent Sibling Combinator
          • Uses the plus (+) sign to combine sequence of selectors.
          • It selects elements that are exactly after the element indicated before it.
          • Example:

            div + p targets all p elements which are exactly after the ending tag of thediv element.

        2. General Sibling Combinator
          • Uses the tilde (~) to combine sequence of selectors.
          • It selects all elements that follows the element indicated before it as long as they share the same parent.
          • Example:

            div ~ p targets all p elements that follows after the div element even if they are not exactly after the div element.

  • Pseudo-Classes
    • These are used to target elements that has a changed state typically from a user interaction.
    • It is indicated by a colon (:)
    • Types of Pseudo-Classes:
      1. Dynamic Pseudo-Class
        1. link-pseudo-class
          • Example: :link, :visited
        2. user action pseudo-class
          • Example: :hover, :active, :focus
      2. Target Pseudo-Class
        • It is usually associated with the href attribute.
        • Example: :target
      3. Language Pseudo-Class
        • It specifies the way of presenting text.
        • Example: :lang()
      4. UI Element States
        • It is commonly used with forms.
        • Example: :enabled, :disabled, :checked, :indeterminate
      5. Structural Pseudo-Class
        • It is associated with elements in the document tree.
        • Example: :root, :first-child, :only-child, :nth-child()
      6. Negation
        • It negates the specified value for the pseudo-class.
        • Example: :not()
    • Pseudo-Elements
      • In CSS3, it is indicated by a double colon (::).
      • Example:
        • :first-letter
        • ::before, ::after
